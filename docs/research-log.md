## Local diagnostic notes

- `2026-03-14`: The original Veil bootstrap used `git clone --filter=blob:none --depth 1`, which succeeded in fetching commit and tree metadata but failed later during `git checkout` when Git attempted to lazily fetch blobs from the promisor remote. The observed failure was `Could not resolve host: github.com` while fetching a promised object.
- `2026-03-14`: A direct `git ls-remote https://github.com/mozilla-firefox/firefox HEAD` succeeded but took about 33 seconds in this environment, indicating GitHub access is available but slow and therefore a poor fit for deferred blob fetches.
- `2026-03-14`: Downloading a pinned `codeload.github.com` source archive for commit `16a7dee3ca6fb0967cd84314393ad1d2e213e4a6` completed reliably and produced a usable extracted source tree.
- `2026-03-14`: `build/moz.configure/util.configure` shows `project_flag()` values are only settable via `imply_option`, which means desktop Firefox product defaults for `MOZ_SERVICES_HEALTHREPORT` and `MOZ_NORMANDY` must be changed in `browser/moz.configure` rather than in a standalone mozconfig.
- `2026-03-14`: Firefox configure initially failed the WASI linker probe because the system `clang` still searched `/usr/lib/clang/21/...` for wasm builtins even when Veil supplied a separate `WASI_SYSROOT`. Extracting a matching local `clang` package into `upstream/toolchain/root` fixed the resource-dir mismatch and allowed `./mach configure` to complete.
- `2026-03-14`: Current upstream `third_party/rust/glslopt` conflicts with glibc 2.43 C11 thread definitions (`once_flag`, `call_once`) in this environment. Veil works around that by path-patching the vendored crate and guarding the duplicate definitions in `threads_posix.h`.
- `2026-03-14`: Current Clang 21 emits `-Wimplicit-int-conversion-on-negation` against generated Servo style constants and treats it as an error under Firefox's warning policy. Veil's current Linux build attempt uses `CFLAGS` and `CXXFLAGS` to add `-Wno-error=implicit-int-conversion-on-negation` as a host-toolchain workaround pending an upstream or source-level fix.
